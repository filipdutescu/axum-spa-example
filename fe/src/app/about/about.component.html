What is this project?

This project is made with the aim to provide an example of how one can build a web application using a SPA framework (such as Angular) and Axum for the API server.

It is a basic IRC-type web application, which supports creating and joining rooms, in which one can talk to multiple users at a time. History is not preserved, thus, if a user leaves the chat, the messages received up until that point will be lost.

How is it made?

The following software was used throughout the project, with the aim of being a guideline or reference to others who wish to use Axum when creating a rich web app:

Axum The backend framework used.

This is the backend framework used to create the REST API used by the frontend you are seeing now. It is highly flexible and customizable which represents great advantages. To learn more about it, please take a look at their GitHub repository and their documentation.

Angular The frontend framework used.

This is the frontend framework used to create the UI. It is robust, mature and used by many organizations. To learn more about it, visit their website. To help with the creation of the UI, Angular Material was used.

Docker and Docker Compose The virtualization software used for development and deployment.

This is used to help containerize the two services, in order to ease deployment and make testing out the product a breeze, removing the need to install all of its dependencies manually. Learn more about Docker on their website.
